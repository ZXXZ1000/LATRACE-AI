# LATRACE v0.1.0

The first public open-source release of LATRACE, formerly OmniMemory.

## What LATRACE is

LATRACE is a production-ready memory layer for AI applications that need more than simple vector recall. It combines:

- hybrid retrieval across vector, graph, and keyword search
- temporal reasoning over evolving user and world state
- multi-tenant isolation for SaaS-style deployments
- self-hostable infrastructure built around FastAPI, Qdrant, Neo4j, and PostgreSQL/SQLite

## Who this release is for

- teams building AI assistants that need memory across sessions
- developers replacing lightweight agent memory with a more structured backend
- product teams that need tenant-aware memory and auditability
- researchers and builders who want to reproduce benchmark and showcase flows locally

## What is included in v0.1.0

- the standalone LATRACE workspace with `modules/memory` as the main public service
- Docker Compose for local self-hosted setup
- CI for linting and test checks on pull requests and pushes to `main`
- GHCR image publishing for the memory service
- public docs for API reference, ADK integration, and tenant isolation
- benchmark entry points for LoCoMo, LongMemEval, showcase, and archived runs

## Quick start

### Self-host locally

```bash
git clone https://github.com/ZXXZ1000/LATRACE.git
cd LATRACE
cp .env.example .env
docker compose up --build
```

### Run locally without Docker

```bash
uv sync
cp .env.example .env
uv run python -m uvicorn modules.memory.api.server:app --host 0.0.0.0 --port 8000
```

## Known limitations

- public docs are still being consolidated; some older internal design material has not been republished as polished external docs yet
- example starter apps are still limited and will expand after `v0.1.0`
- production deployments still require you to configure external dependencies and provider credentials
- benchmark reproduction may require additional dataset familiarity and environment setup beyond the quick start path
